关于vue生命周期的理解
创建前后 加载前后 更新前后 销毁前后
创建前后
beforeCreated阶段,vue实例的挂载元素$el和数据对象都为undefined,还未初始化，在create阶段，vue实例的数据对象data有了，$el还没有

加载前后
beforeMount阶段，vue实例的$el和data都初始化了，但还是挂载之前的虚拟dom节点，data.message还未替换，在mounted阶段，vue实例挂载完成，data.message成功渲染

更新前后
当data变化时，会触发beforeUpdate和updated方法

销毁前后
在执行destroy方法后，对data的改变不会再出发周期函数，说明此时vue实例已经解除了事件监听以及和dom的绑定，但是dom结构依然存在

vue的ajax请求代码
请求的数据是不是要被其他组件公用，仅仅在请求的组件内使用，就不需要放入vuex的state里，如果需要被其他地方复用，而且是很大几率上是需要，将请求放入action里，方便复用

restful API
api的标准，无状态请求，请求的路由地址是固定的，如果是tp5则先路由配置中把资源路由配置好，标准有.get .post .delete .put

ajax返回error的原因
1.dataType错误
后台返回的dataType类型和前台写的不一致会跳入error
json格式错误也会跳入error

2.data不能不写
data为空也一定要穿”{}”,不然返回的xml格式的

3.传递的参数
必须是ajax支持的格式

4.URL路径问题
路径不能有中文

call() 和apply()的作用和区别
call() 和apply()都可以用来间接调用函数，都可以显示调用所需要的this，即任何函数都可以作为任何对象的方法来调用，两个方法都可以指定调用实参

区别：
call()和apply()的基本区别：在于将参数传递给函数
call():使用其自由的实参列表作为函数的参数
apply：要求以数组的形式传入参数
