代理

代理与网关的区别

严格说，代理连接的是两个或多和使用相同协议的应用程序，而网关连接的是两个或多个使用不同协议的端点。网关扮演的是“协议转换器”的角色，即使客户端和服务器使用的是不同的协议，客户端也可以通过它完成与度武器之间的事务处理

实际上，代理和网关的区别很模糊，由于浏览器和服务器实现的是不同版本的HTTP，代理也经常要做一些协议转换工作，而商业化的代理服务器也会实现网关的功能来支持SSL安全协议，SOCKS防火墙、FTP访问，以及基于web的应用程序。

代理的作用

用途 举例
安全过滤 不良内容不宜给儿童
文档访问控制 不同用户使用权限问题
安全防火墙 代理服务器会在网络中的单一单圈节点上限制哪些应用层协议的数据可以流入或者流出一个组织，还可以对流量进行详细的检查（消除病毒web和email使用的挂钩程序）
web缓存 代理缓存维护了常用文档的本地副本，并将它们按须提供，以减少缓慢而又昂贵的因特网通信
反向代理 代理可以假扮web服务器。这些被称为代替物或反向代理的代理接受发给Web服务器的真实请求，但与web服务器不同的是他们可以发起与其他服务器的通信，以方便按需定位所请求的内容。可以用这些反向代理来提高访问慢速Web服务器上公共内容时的性能。在这种配置中，通常将这些反向代理称为服务器加速器
内容路由器 根据因特网流量状况以及内容类型将请求导向特定的web服务器。内容路由器也可以用来实现各种服务器的请求

代理既可以处理代理请求，也可以处理服务器请求

使用完整和部分url的规则如下：

  • 如果提供的是完整的URL，代理就应该使用这个完整的URL
  • 如果提供的是部分URL，而且有host首部，就应该用host首部来确定原始服务器的名字和端口号
  • 如果提供的是部分URL，而且没有host首部，就应该用其他方法来确定原始服务器：
    1. 如果代理是代表原始服务器的替代物，就可以用真实服务器的地址和端口号来配置代理；
    2. 如果流量被拦截了，而且拦截者也可以提供原始的ip地址和端口号，代理就可以使用拦截技术提供的ip地址和端口号
    3. 如果所有方法都失败了，代理没有足够的信息来确定原始服务器，就必须返回一条错误报文（通常是建议用户升级到支持host首部的浏览器）

缓存的处理过程

http get报文的基本缓存处理过程：

  • 接受————缓存从网络中读取抵达的请求报文
  • 解析————缓存对报文进行解析，提取出URL和各种首部
  • 查询————缓存查看是否有本地副本可用，如果没有，就获取一份副本（并保存在本地
  • 新鲜度检测————缓存查看已缓存副本是否足够新鲜，如果不是就向服务器询问是否有更新
  • 创建响应————缓存会用新的首部和已缓存的主体来构建一条响应报文
  • 发送————缓存通过网络将响应发回给客户端
  • 日志————缓存课选的创建一个日志文件条目来描述这个事务

设置缓存控制

控制Apache的http首部

· mod_headers
通过mod_headers模块可以对单独的首部进行设置。装载了这个模块就可以用设置单个http首部的指令来扩充Apache的配置文件，还可以将这些设置与Apache的常用表达式以及过滤器结合起来一起使用，将这些首部与个别内容关联起来。

· mod_expires
mod_expires模块提供的程序逻辑可以自动启动生成带有正确过期日期的Expires首部。通过这个模块可以将文档的过期日期设置为对其最后一次被访问之后或者其最近修改日期之后的某一时间段。通过这个模块可以为不同的文件类型设置不同的过期日期，还可以使用便捷的详尽描述信息来描述其缓存能力

· mod_cern_mata
通过mod_cern_meta模块可以讲一个包含黄体酮首部的文件与特定的对象联系起来。启动这个模块时，就创建了一组“元文件”，每个需要控制的文档一个，而且还为每个元文件添加所期望的首部

通过HTTP_EQUIV控制HTML缓存

http服务器响应首部用于回送文档的到期信息以及缓存控制信息。web服务器与配置文件进行交互，为所能提供的文档分配正确的Cache-Control首部
